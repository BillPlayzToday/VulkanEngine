                                  Raymond Wood Alexander Overvoorde 10 months ago

                                  I am using Ubuntu v20.04, and the following may be helpful for others using Ubuntu.

                                  Because my Dell E6530 laptop uses an older GPU for which up to date drivers for Ubuntu are not available, I used Software & Updates to select the default nouveau driver (which emulates vulkan support in software). Afterwards, vkcube renders the expected spinning cube, but also reports a non-fatal warning indicating MESA does not provide full vulkan support. Also, after I installed an OLDER vulkan sdk, VK_LUNARG_standard_validation was found but VK_KHRONOS_validation was not.

                                  The fix was to uninstall the older vulkan sdk, and then install the newer vulkan sdk as follows, after which nothing else was necessary:

                                  sudo apt-get remove vulkan-utils
                                  wget -qO - http://packages.lunarg.com/... | sudo apt-key add -
                                  sudo apt update
                                  sudo apt install vulkan-sdk

                                  After that, vkcube stills reports the non-fatal MESA support warning, but my implementation of the vulkan-tutorial code correctly finds the newer VK_KHRONOS_validation layers.

                                                                                                                                Mochii Moth Kyle Pearce a year ago

                                                                                                                                I had the same problem, turns out the problem was an old version of Vulkan forgot to clean up behind itself when it was uninstalled, and left stuff in the registry.

                                                                                                                                Here's how you can fix it (if you have the same problem I had) :
                                                                                                                                1. In your Vulkan install folder, in the tool subfolder, run vkconfig.exe
                                                                                                                                2. In the tool menu (top left) chose "Vulkan installation analysis"
                                                                                                                                3. Click on "Vulkan Explicit Layers"
                                                                                                                                4. If there's errors about .json files, you probably have the same problem as I.
                                                                                                                                5. Use regedit and go to "HKEY_LOCAL_MACHINE\SOFTWARE\Khronos\Vulkan\ExplicitLayers"
                                                                                                                                6. Delete all entries corresponding to the problematic lines in the "Vulkan installation analysis"

                                                                                                                                In my case, I had to delete all lines that pointed to a different version than the one currently installed.

                                                                                                                                                For anyone having trouble with the "VK_LAYER_KHRONOS_validation" part, it will not work unless you have the proper path set to the "Bin" folder in the "VulkanSDK" folder.

                                                                                                                                                To make sure it's set up correctly, open the "vkconfig" app by searching "Vulkan Configurator" in the windows home search bar. When this is open, go to the second tab (the "Layer Manager" tab).

                                                                                                                                                If there is a bunch of "HKEY..." paths in the top box, it is not set to the right path. To select the correct path, click the "Use custom layer paths" check box above the path box, so that it is marked. This should 'erase' (don't worry they aren't gone) the "HKEY..." paths.

                                                                                                                                                Next, click the "Add" button next to the box and navigate to the "VulkanSDK" folder and then find the "Bin" folder. Do not open the "Bin" folder, just click it, and click "Select Folder", to set the path.

                                                                                                                                                BEFORE YOU EXIT the configurator click the checkbox labeled "Undo changes on exit" so that it is unmarked. I repeat, it should be UNmarked. Then click "Save" and you can exit the configurator now.

                                                                                                                                                Now try running the program again and the "runtime_error" that was thrown before should be gone. Let me know if you have any problems!
